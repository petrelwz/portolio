Atos is an IT services company that provides various IT solutions to clients. Among Atos offerings, Atos OneContact is a user support solution for clients so that their employees could reach their service desk through mobile devices.

Opportunity

The current user support solution is that users need log in an online web portal to raise a support ticket. The solution is problematic in the following ways:

As part of the digital workplace, our team started to transform the current user support solution into a mobile based service desk.

My Role

I’m the lead designer of this project, working closely with the PMs and engineers to design the scope of this project. I worked on sketches, prototypes, visual designs and user tests.

Process

We’ve already had an app running. I faced some difficulties in pushing design changes. In order to get the management / technical team on board, I need to use some techniques to persuade them.

- Focus on explaining clearly the benefit of the changes when I present my design solution.

- Communicate and identify problems with the 7-8 users pilot, so that I have more data to support my design.

- Push design changes in different waves

  • Wave 1: Pushing small changes and collect feedback
  • Wave 2: Pushing changes that require more resources

Wave 1

Four small changes are proposed in Wave 1

Benefit
  • Saves one click in choosing a topic
  • Know what are the options at one glance
  • Reduce error - no need to see the pop up message when not selecting an issue
Trade-off
  • User cannot use “cancel” button to go back before they select an issue, they could only use “back” button to go back to the previous page.
Benefit
  • Made notification possible
  • Different tabs ( news, disruptions, maintenance) are always available on the screen, so no need to fold/unfold
  • Reduce scrolling
Benefit
  • Save clicks
  • Calendar is available at one glance, so users know what their options are
  • Reduce error by offering “OK” buttons only when needed
Benefit
  • After confirmation - an action is done, the page goes back to “Contact”. Otherwise, users need to click “back” button again.
  • Give users transparency — could provide more information about the information being confirmed e.g. time, date.
